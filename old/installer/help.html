Inledning

Bandbreddstestet TPTEST 5.0 ersätter TPTEST Statistik, som tas ur drift. Version 5.0 innehåller dels ett förenklat gränssnitt, dels mer avancerade mätfunktioner. Nu finns även möjligheten att skapa rapporter och göra flera olika mätningar. Bruksanvisningen går igenom alla olika typer av mätningar och dess inställningar.

Denna version av TPTEST 5.0 går att använda på datorer med Windows OS 2000 och XP.

Om du har en specifik fråga, sök i Frågor & Svar eller på TPTESTs användarforum; www.forum.tptest.se. Om du inte hittar svar på din fråga, skicka ett mail till bandbreddstest@iis.se och beskriv ditt problem så noga som möjligt.

Rapport

Om du vill skapa en rapport, gå till ”Arkiv” och välj "Skapa rapport". Rapporten kan du sedan skicka till supportavdelningen hos din operatör, om du behöver hjälp eller om något verkar vara fel.

För att kunna skicka rapporten måste du antingen kopiera och klistra in informationen i ett e-postmeddelande, eller spara filen på din dator och sedan bifoga filen. Rapporten kan inte skickas automatiskt.

För att kunna skicka rapporten måste du antingen kopiera och klistra in informationen i ett e-postmeddelande, eller spara filen på din dator och sedan bifoga filen. Rapporten kan inte skickas automatiskt.

Viktigt!

Rapporten visar bland annat vilka servrar din dator för tillfället är uppkopplad mot. Det kan vara webbplatser som aftonbladet.se eller dn.se, men även webbplatser som man inte vill visa att man besöker, till exempel ett politiskt parti. För att undvika att få med sådan information bör man stänga ned webbläsaren, fildelningsprogrammet, e-post-programmet, msn och annat som berör Internet innan man skapar rapporten. När den är klar kan man börja surfa igen!

Tillgänglighet
I detta sammanhang används begreppet tillgänglighet för ”hur väl anslutningen fungerar”. Testen för tillgänglighet mäter alltså inte hur snabbt det går att skicka data över Internet, utan kontrollerar i stället att all data kommer fram. Om data inte når sin slutstation, innebär det att datorn gör fler försök, så kallade omsändningar, innan den till sist ger upp. Även om datorn ifråga aldrig når ”ge upp-stadiet”, kommer hastigheten att minskas betydligt.

Under ”Tillgänglighet” finns tre val:
- Resultat som graf
- Resultat som lista
- Inställningar

Resultat som graf
Här visas resultatet av dina mätningar i ett diagram, där kurvan visar 0 till 100 procents tillgänglighet. Ju fler servrar du testar mot, desto rättvisare blir resultatet.

OBS! Det är inte alla servrar som svarar på mätningarna icmp echo (ping), webservertrafik eller ftp. Om du försöker testa mot en server som inte svarar på den typen av mätningar kommer testet att ta mycket längre tid och TPTEST kommer att ange denna typ av test som misslyckat. För att veta kan man testa att ansluta med en vanlig klient. Om det inte går att ansluta med en vanlig klient kommer det inte gå att ansluta med tptest heller.

Resultat som lista
Här ser du samma information som i grafen, men i en lista.

Inställningar
Under ”Inställningar” listas de servrar du valt i serverlistan. (För att lägga till nya, skriv ”Alt+s”). Genom att markera en eller flera serveradresser kan du välja vilken eller vilka servrar som ska vara aktiverade under testet, och vilken typ av test du vill göra.

Det finns två typer av test, ICMP och TCP.
ICMP står för Internet Control Message Protocol. Här genomför testprogrammet en så kallad ping och väntar på svaret (pong). Testet fungerar så att ett datapaket skickas iväg, och när servern får paketet skickar den tillbaka ett paket som ser likadant ut. När TPTEST ser att paketet kommit tillbaka räknas testet som lyckat. Kommer inget paket tillbaka räknas det som misslyckat.

TCP står för Transfer (eller Transmission) Control Protocol
TCP-testet fungerar lite annorlunda. TPTEST-programmet kopplar då upp sig via TCP-protokollet mot det system som är valt i serverlistan och genomför en så kallad TCP-handskakning. När denna är klar säger TPTEST "tack för mig" och skickar FIN-flaggan (finish), som markerar att testet har lyckats. Om ”hälsningen” misslyckas kommer testet att markeras som misslyckat.

Exempel:
Vi antar att du vill testa din uppkoppling mot www.aftonbladet.se.
Först trycker du ”Alt+s” för att få upp serverlistan, sedan fyller du i informationen:

adress: www.aftonbladet.se
tcp-ping-port: 80
välj: ”Lägg till”
Nu kommer www.aftonbladet.se att dyka upp i listan. Har du aktiverat TCP-mätningen, kommer TPTEST vid nästa tillgänglighetstest att kontrollera om det går att nå www.aftonbladet.se.

OBS: En del brandväggsinställningar kan medföra att TPTEST inte lyckas med testet.
Systemet kan gå att nå via andra portar, men om just den port du har valt att testa mot blockeras av brandväggen kommer testet att räknas som misslyckat.

Svarstider och paketförluster

Under denna flik finns fem underflikar.
-Svarstider som graf
-Paketförluster som graf
-Jitter som graf
-Resultat som lista
-Inställningar

I detta test skickas en mängd paket till olika platser på Internet. Dels räknas svarstiden för varje server, dels tiden det tar för paketen att komma tillbaka. Paketen som skickas är även här av typen icmp echo (ping).

Denna mätning är intressant för dig som spelar spel över Internet, där höga paketförluster eller svarstider kan resultera i att din dator ”släpar efter”. Det brukar kallas att man får lagg, eller till och med blir "link dead", det vill säga att spelet kastar ut dig.

Svarstider som graf
Denna flik ritar upp svarstiderna över en tidsaxel. Låga svarstider är bra, medan höga är dåligt (=gör att det går långsamt).

Paketförlust som graf
Denna flik visar datorns paketförluster som en kurva. Paketförluster är något man vill ha väldigt lite av, då Internetförbindelsen kan fungera väldigt oförutsägbart och ibland bli helt oanvändbar om paketförlusterna blir för höga. Resultatet visas i procent.

Jitter som graf
Jitter kallas de variationer eller fördröjningar som kan uppstå på en data- eller telefoniförbindelse, även kallat ”digitalt brus”.

Resultat som lista
Här visas testresultatet som en lista.

Inställningar
Här ställer du in vilka servrar du vill testa mot. Det kan vara intressant att testa flera olika servrar, både i och utanför Sverige. Notera att även här kan brandväggar ställa till det och ge orättvisa resultat, genom att helt enkelt inte svara när TPTEST-programmet frågar om systemet går att nå.

Viktigt!

Brandväggen behöver inte nödvändigtvis vara den du har hemma, utan kan stå precis framför servern du försöker nå. Koppla aldrig ur eller stäng av en brandvägg i hemmet om du inte är fullt medveten om riskerna och vad detta innebär.

Genomströmning

I detta test mäts hur snabbt data skickas från olika servrar. Detta är intressant för den som laddar ned mycket filer.

Här finns sex flikar:
- TPTEST nedströms graf
- TPTEST upptröms graf
- Http som graf
- Ftp som graf
- Resultat som lista
- Inställningar

TPTEST nedströms graf
Här visas en kurva över hur snabbt det går att ladda ner data till din dator. Om du laddar hem mycket data är det alltså detta test du ska använda. Mätningen här är av typen TPTEST.

TPTEST uppströms graf
Här visas en kurva över hur snabbt det går att ladda upp data från din dator till en server. Om du laddar upp mycket data, kanske skickar väldigt stora mejl eller gör stora säkerhetskopior över nätet är detta mycket intressant. Mätningen här är av typen TPTEST.

Http som graf
Detta visar hur snabbt det går när du hämtar information över http-protokollet, det vill säga samma protokoll som används för vanlig surfning.

Ftp som graf
Här visas en kurva över hur snabbt det går när du hämtar information över ftp-protokollet.

Resultat som lista
Här visas testresultatet som en lista.

Inställningar
Här visas alltså serverlistan, som du kan redigera med ”Alt+s”. Längst ner har du fyra checkboxar som du kan fylla i. Dessa är till för att avgöra vilken typ av mätning som ska genomföras mot servern. Alternativen är:
- aktiverad - är denna förkryssad kommer en mätning göras mot servern
- tptest - är denna förkryssad kommer en tptestmätning göras mot servern
- http - är denna förkryssad kommer en fil hämtas över http
- ftp - är denna förkryssad kommer en fil hämtas över ftp

Verktyg

Den sista fliken i det avancerade läget innehåller några mer avancerade verktyg:
- TPTEST 3
- ifconfig
- netstat
- ping
- traceroute

Dessa testverktyg är främst avsedda för specialister, supportavdelningar och andra som är insatta i hur de fungerar.

TPTEST 3
TPTEST 3-fliken ger dig möjlighet att skräddarsy en TPTEST-mätning.

ifconfig
Denna flik kommer att visa allt som går att få reda på om konfigureringen för din anslutning till Internet.

netstat
netstat-fliken visar vilka portar som systemet lyssnar på, vilka system det kommunicerar med, samt statusen för dessa uppkopplingar.

ping
Här kan du skräddarsy beteendet för ping, eller icmp echo.

traceroute
traceroute är ett verktyg för att ta reda på vilken väg din dator tar över Internet för att nå en annan dator.

OBS! När du mäter med de testtyper som är Microsoftstyrda, som http, ping och traceroute, fungerar inte svenska tecken (å ä ö).
